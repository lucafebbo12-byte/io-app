# CLAUDE.md

## Project collaboration rules
- Ask clarifying questions before coding when requirements are ambiguous.
- Prefer TypeScript over JavaScript unless explicitly requested otherwise.
- Propose a plan first for non-trivial tasks.
- Keep changes small and easy to review.
- Run tests/lint after edits when available and report results.
- Do not rewrite unrelated files.
- Suggest a git commit point after each stable milestone.

## Working with Codex + Claude
- If Codex already modified files, inspect diffs before editing.
- Continue existing patterns instead of restyling/refactoring everything.
- Explain tradeoffs briefly and recommend one option.
- For risky changes, ask for confirmation before proceeding.
